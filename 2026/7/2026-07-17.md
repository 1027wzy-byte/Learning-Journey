# 一、为什么需要集合？

数组存在三个明显缺陷：

- 长度固定，无法自动扩容
- 插入、删除元素效率低
- 无法自动去重

Collection 提供了统一的数据容器，可以自动扩容、增删元素，并提供丰富 API。

## ArrayList

### 底层结构

```text
Object[]
```

### 特点

优点：

- 查询速度快 O(1)
- 内存利用率高

缺点：

- 中间插入、删除需要移动元素

自动扩容：

```text
容量 × 1.5
```

适用场景：

> 读多写少（绝大多数业务）

### 常用方法

```java
add()
add(index, e)
get(index)
set(index, e)
remove(index)
contains()
size()
clear()
```

### 课堂代码

```java
List<String> list = new ArrayList<>();

list.add("Apple");
list.add("Banana");
list.add("Cherry");

System.out.println(list.get(0));

list.remove(0);

list.clear();
```

# 六、PriorityQueue（优先队列）

普通队列：

```text
先来先服务
```

PriorityQueue：

```text
按优先级出队
```

默认：

数字越小，优先级越高。

示例：

```java
PriorityQueue<Integer> queue = new PriorityQueue<>();

queue.offer(5);
queue.offer(1);
queue.offer(2);

System.out.println(queue.poll());
```

输出：

```text
1
2
5
```


|---------|------|

## HashSet

底层：

```text
HashMap
```

去重流程：

```text
hashCode()

↓

equals()
```

只有：

- hashCode 相同
- equals 返回 true

才认为重复。

因此：

> 自定义对象必须同时重写 **hashCode()** 和 **equals()**。

---

```text

```

核心代码：

```java
Set<Character> set = new LinkedHashSet<>();
```

# 八、Iterator（迭代器）

统一遍历集合。

基本写法：

```java
Iterator<String> it = list.iterator();

while (it.hasNext()) {
    String str = it.next();
    System.out.println(str);
}
```

增强 for：

```java
for (String str : list) {

}
```

底层也是 Iterator。

# 九、ConcurrentModificationException

错误示例：

```java
for (String s : list) {
    list.remove(s);
}
```

运行会抛出：

```text
ConcurrentModificationException
```

原因：

遍历过程中直接修改集合。

正确写法：

```java
Iterator<String> it = list.iterator();

while (it.hasNext()) {
    String s = it.next();

    if (s.equals("Go")) {
        it.remove();
    }
}
```

遍历删除元素时必须使用：

```java
iterator.remove();
```
