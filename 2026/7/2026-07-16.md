## 今天最大的收获

### 1. 理解了 String 为什么不可变

以前只知道 String 不能修改，但今天知道了真正原因：

- String 类被 `final` 修饰，不能继承。
- 内部保存数据的数组使用 `private final` 修饰。
- 所谓"修改字符串"，实际上是创建了新的 String 对象，再让变量指向新的对象。

例如：

```java
String s = "Java";
s = s + "17";
```

真正变化的是变量 `s` 的指向，而不是 `"Java"` 这个对象本身。

---

### 2. 理解了字符串常量池

以前一直不理解为什么：

```java
String a = "hello";
String b = "hello";
```

使用 `==` 会返回 true。

现在知道：

字面量创建的字符串会先去字符串常量池查找，如果已经存在，就直接复用对象，因此两个变量指向的是同一个对象。

而：

```java
new String("hello");
```

无论常量池有没有，都会重新创建新的对象，所以地址一定不同。

---

### 3. 明白了为什么字符串比较不能使用 ==

今天最大的印象就是：

```
== 比较地址
equals() 比较内容
```

以后字符串比较内容，一律使用：

```java
equals()
```

不要再使用 `==`。

---

### 4. 学会了 StringBuilder

今天终于知道为什么很多人说：

```
循环拼接字符串不要使用 +
```

因为 String 不可变，每拼接一次都会创建新的对象。

例如：

```java
result += i;
```

实际上不断创建新的 String。

正确写法：

```java
StringBuilder sb = new StringBuilder();

sb.append(...);
```

效率高很多。
